**文档生成时间：** 2026 年 3 月 10 日

## 一、流程概览与智能体定位

### 1.1 流程阶段划分

| 阶段 | 主要活动 |
|------|----------|
| 产品规划 | 产品提案、需求评审、ID 设计、PRD 编写 |
| 项目立项 | 制定计划、项目立项、PRD 确认 |
| 设计开发 | 结构设计、硬件设计 (SCH/PCB)、软件设计、开模/手板 |
| EVT 阶段 | EVT 试产、EVT 检验、问题优化、EVT 评审 |
| DVT 阶段 | DVT 试产、DVT 验证、问题优化、DVT 评审 |
| PVT/MP 阶段 | PVT 试产、PVT 验证、MP 爬坡、项目结项 |

### 1.2 智能体角色定位

| 智能体角色 | 核心职责 | 介入阶段 |
|------------|----------|----------|
| 项目助理智能体 | 进度跟踪、任务提醒、文档管理 | 全流程 |
| 评审智能体 | 评审材料检查、评审记录、问题追踪 | 各评审节点 |
| 问题跟踪智能体 | 问题列表管理、状态更新、超时预警 | EVT/DVT/PVT |
| 文档智能体 | 文档生成、归档、版本管理 | 全流程 |

```

           │          │

```

**问题优先级定义：**

| 级别 | 定义 | 响应时间 | 处理时间 |
|------|------|----------|----------|
| P0-致命 | 阻塞项目进度，无法继续 | 2 小时 | 24 小时 |

| P2-一般 | 影响次要功能 | 1 天 | 1 周 |
| P3-建议 | 优化建议 | 3 天 | 迭代版本 |

## 四、关键工作流实现

### 4.1 阶段转换规则

**EVT → DVT：**

前置条件：
- EVT 评审通过
- 无 P0/P1 开放问题
- 硬件设计定型
- 软件版本稳定

自动动作：
- 创建 DVT 问题跟踪表
- 更新项目状态
- 发送 DVT 启动通知
- 安排 DVT 试产

**DVT → PVT：**

前置条件：
- DVT 评审通过
- 模具验收完成
- 认证启动
- 无 P0 开放问题

自动动作：
- 创建 PVT 问题跟踪表
- 更新项目状态
- 发送 PVT 启动通知

**PVT → MP：**

前置条件：
- PVT 评审通过
- 软硬件稳定
- 直通率达标
- 遗留问题满足质量要求

自动动作：
- 更新项目状态为 MP
- 发送 MP 启动通知
- 安排生产爬坡

### 4.2 Cron 任务配置

**每日项目提醒：**
```json
{
  "id": "project-daily-reminder",
  "name": "项目每日提醒",
  "schedule": "0 9 * * 1-5 (工作日 9:00)",
  "功能": [
    "读取项目状态表",
    "检查今天到期的任务/评审",
    "检查超时未关闭的问题",
    "发送提醒给相关负责人"
  ]
}
```

**问题超时检查：**
```json
{
  "id": "issue-overdue-check",
  "name": "问题超时检查",
  "schedule": "0 9,15 * * 1-5 (工作日 9:00 和 15:00)",
  "功能": [
    "检查截止日期已过但未关闭的问题",
    "检查 P0/P1 问题处理超时",
    "发送预警通知"
  ]
}
```

## 七、智能体协同机制

### 7.1 阶段 - 智能体映射

| 阶段 | 主导智能体 | 协同智能体 |
|------|------------|------------|
| 产品规划 | 产品智能体 | 项目智能体 |
| 项目立项 | 项目智能体 | 产品智能体 |
| 设计开发 | 硬件/软件/结构智能体 | 项目智能体、测试智能体 |
| EVT | 测试智能体 | 项目智能体、硬件/软件/结构智能体、工厂智能体 |
| DVT | 测试智能体 | 项目智能体、硬件/软件/结构智能体、工厂智能体 |
| PVT | 测试智能体 | 工厂智能体、项目智能体 |
| MP | 工厂智能体 | 项目智能体 |

### 7.2 数据流转

```
产品智能体 ──PRD──▶ 硬件智能体
                   ──PRD──▶ 软件智能体
                   ──PRD──▶ 结构智能体
                   ──PRD──▶ 包装智能体
                   
项目智能体 ──Schedule──▶ 所有智能体

硬件智能体 ──BOM/PCB──▶ 结构智能体
            ──GPIO──▶ 软件智能体
            
结构智能体 ──PCB 板框──▶ 硬件智能体

测试智能体 ──问题列表──▶ 硬件/软件/结构智能体

工厂智能体 ──试产报告──▶ 项目智能体
            ──问题列表──▶ 测试智能体
```

### 7.3 通知机制

| 事件类型 | 触发智能体 | 通知对象 | 通知内容 |
|----------|------------|----------|----------|
| 评审通过 | 项目智能体 | 相关负责人 | 评审结果、下一步任务 |
| 问题超时 | 测试智能体 | 问题责任人、项目经理 | 问题详情、超时天数 |
| 阶段完成 | 项目智能体 | 所有负责人 | 阶段总结、下一阶段计划 |
| 物料缺料 | 工厂智能体 | 采购、项目经理 | 缺料清单、影响评估 |
| 设计变更 | 变更智能体 | 相关负责人 | 变更内容、影响范围 |

## 八、实施优先级建议

### 8.1 第一优先级（核心）

| 智能体 | 理由 |
|--------|------|
| 项目智能体 | 贯穿全流程，协调所有智能体 |
| 测试智能体 | EVT/DVT/PVT 核心环节，问题管理关键 |
| 硬件智能体 | 硬件开发是研发核心 |

### 8.2 第二优先级（重要）

| 智能体 | 理由 |
|--------|------|
| 产品智能体 | 需求管理是源头 |
| 结构智能体 | 结构设计关联模具、成本 |
| 工厂智能体 | 试产/量产是产出环节 |

### 8.3 第三优先级（完善）

| 智能体 | 理由 |
|--------|------|
| 软件智能体 | 软件迭代快，可后期完善 |
| 包装智能体 | 包装相对独立，影响范围小 |
