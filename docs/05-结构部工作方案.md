# 研发工作流程智能体落地 - 结构部工作方案

**版本：** V1.0  
**编制日期：** 2026 年 3 月 11 日  
**编制部门：** 结构部

---

## 一、概述

### 1.1 背景

根据《研发工作流程智能体搭建方案》，结构智能体负责 ID 设计、结构设计、模具开发、包装设计等关键职能。本文档明确结构部在智能体落地过程中需要完成的具体工作内容、交付物和时间节点，重点解决结构设计管理、与硬件部尺寸配合、模具开发跟踪、结构问题流转的核心需求。

### 1.2 核心需求

**结构智能体核心能力：**
- ID 设计评审支持与方案管理
- 结构设计与硬件尺寸联动
- 模具开发进度跟踪与预警
- 包装设计管理
- 结构问题接收与修复跟踪
- 与测试部的问题流转联动

### 1.3 结构部职责定位

| 职责类别 | 具体内容 | 对应智能体功能 |
|----------|----------|----------------|
| ID 设计 | 产品外观设计、渲染图、CMF 方案 | ID 设计管理功能 |
| 结构设计 | 3D 建模、2D 图纸、DFM 分析 | 结构设计管理功能 |
| 模具管理 | 模具开发、试模、验收 | 模具跟踪功能 |
| 包装设计 | 包装方案、包材设计、运输测试 | 包装管理功能 |
| 手板管理 | 手板制作、验证、评审 | 手板管理功能 |

## 四、数据同步机制

### 4.1 结构数据同步架构

```
结构数据输入层（飞书对话/设计文档/评审记录）
         ↓
结构主数据存储（唯一数据源 - 结构设计主表）
         ↓
字段映射引擎（检测变更→查找文档→定位位置）
         ↓
文档同步服务（ID 方案│结构规格书│模具管理表│包装文档）
         ↓
跨部门联动（硬件部尺寸同步、测试部问题流转）
```

### 4.2 与硬件部的数据联动

结构部与硬件部共享尺寸和空间数据：

| 结构部字段 ID | 硬件部字段 ID | 字段名称 | 同步方向 |
|---------------|---------------|----------|----------|
| STR007 | HW_尺寸限制 | 外形尺寸 | 双向同步 |
| STR_空间约束 | HW_器件布局 | 空间约束 | 结构部→硬件部 |
| STR_接口位置 | HW_接口定义 | 接口位置 | 双向同步 |
| STR_散热要求 | HW_热设计 | 散热要求 | 双向同步 |

### 4.3 与测试部的问题流转联动

结构部接收测试部分发的结构问题，修复后同步回归测试：

| 数据类型 | 来源 | 目标 | 同步内容 |
|----------|------|------|----------|
| 结构问题 | 测试部 | 结构部 | 问题分配、问题描述 |
| 修复结果 | 结构部 | 测试部 | 修复方案、回归测试请求 |
| 验证结果 | 测试部 | 结构部 | 问题关闭/重新打开 |

## 六、工作流程细化

### 6.1 ID 设计流程

```
需求接收 → 风格确认 → 方案设计 → 内部评审 → 客户确认 → ID 定案
    ↓          ↓          ↓          ↓          ↓          ↓
  产品定位   参考方案   渲染图输出  评审记录   确认签字   ID 冻结
  市场定位   CMF 方案   2D 效果图   修改意见   方案调整   进入结构
```

| 步骤 | 动作 | 输入 | 输出 | 负责人 | 智能体支持 |
|------|------|------|------|--------|------------|
| 1 | 接收需求 | PRD、产品定位 | 设计需求 | ID 设计师 | 自动关联需求 |
| 2 | 风格确认 | 设计需求 | 风格方向 | ID 设计师 | 风格参考库 |
| 3 | 方案设计 | 风格方向 | 渲染图、效果图 | ID 设计师 | 设计模板 |
| 4 | 内部评审 | 设计方案 | 评审记录 | 结构部 | 评审提醒、纪要记录 |
| 5 | 客户评审 | 评审通过方案 | 确认记录 | 客户 | 评审安排 |
| 6 | ID 定案 | 客户确认 | ID 冻结通知 | ID 设计师 | 自动通知相关部门 |

### 6.2 结构设计流程

```
ID 定案 → 结构设计 → DFM 评审 → 手板制作 → 手板验证 → 结构定案 → 开模
    ↓        ↓          ↓          ↓          ↓          ↓         ↓
 ID 文件   3D 建模    可制造性评审  手板下单   装配验证   图纸冻结   模具启动
 外观要求  2D 图纸    成本评估    样件评审   问题修改   BOM 确认   试模安排
```

| 步骤 | 动作 | 输入 | 输出 | 负责人 | 智能体支持 |
|------|------|------|------|--------|------------|
| 1 | 接收 ID 定案 | ID 文件、外观要求 | 结构需求 | 结构工程师 | 自动关联 ID 方案 |
| 2 | 空间评估 | 硬件规格、器件清单 | 空间分配方案 | 结构工程师 | 与硬件部联动 |
| 3 | 结构设计 | 结构需求、空间方案 | 3D 模型、2D 图纸 | 结构工程师 | 设计规范检查 |
| 4 | DFM 评审 | 3D 模型 | DFM 评审报告 | 结构部 + 供应商 | 评审提醒 |
| 5 | 手板制作 | 评审通过模型 | 手板 | 手板厂 | 手板进度跟踪 |
| 6 | 手板验证 | 手板 | 验证报告 | 结构工程师 | 问题记录 |
| 7 | 结构定案 | 验证通过 | 图纸冻结 | 结构工程师 | 自动通知开模 |

### 6.3 模具开发流程

```
开模启动 → 模具设计 → 模具加工 → 试模 → 样件评审 → 修模 → 验收
    ↓         ↓          ↓         ↓        ↓         ↓       ↓
  模具下单   3D 模具设计  CNC 加工   T1 样件   问题记录   修改模具  签字确认
  供应商确认 2D 模具图纸  电火花    试模报告  优化方案   T2 试模   模具入库
```

| 步骤 | 动作 | 输入 | 输出 | 负责人 | 智能体支持 |
|------|------|------|------|--------|------------|
| 1 | 开模启动 | 结构定案图纸 | 模具下单 | 结构工程师 | 自动创建模具记录 |
| 2 | 模具设计 | 3D 模型 | 模具设计方案 | 模具厂 | 设计进度跟踪 |
| 3 | 模具加工 | 模具设计 | 模具 | 模具厂 | 加工进度跟踪 |
| 4 | 试模（T1） | 模具 | T1 样件 | 模具厂 | 试模安排提醒 |
| 5 | 样件评审 | T1 样件 | 评审报告 | 结构部 | 问题记录 |
| 6 | 修模 | 评审问题 | 修改后模具 | 模具厂 | 修模进度跟踪 |
| 7 | 试模（T2/TN） | 修改模具 | 新样件 | 模具厂 | 试模预警（超过 N 次） |

### 6.4 结构问题处理流程

```
问题接收 → 问题分析 → 方案设计 → 方案评审 → 实施修复 → 回归验证
    ↓         ↓          ↓          ↓          ↓          ↓
  测试部分发  问题分类   修改方案   评审记录   修改模具   测试确认
  问题录入   原因分析   受影响分析  成本评估   样件制作   问题关闭
```

| 步骤 | 动作 | 触发 | 责任部门 | 智能体支持 |
|------|------|------|----------|------------|
| 1 | 问题接收 | 测试部分发 | 结构部 | 自动创建问题记录 |
