## ✨ 系统功能

### 🔐 登录注册
- 用户名密码登录
- 完整注册流程（用户名、密码、部门、职务、邮箱、手机）
- 管理员默认账号：`admin` / `Admin@2026`

### 🏠 首页
- **5 个部门智能体入口**：产品部、项目部、硬件部、软件部、结构部
- **项目触发**：输入 PRD 或上传项目提案创建新项目
- **我的项目**：查看所有项目及当前阶段
- **项目流转可视化**：立项 → 规划 → 设计 → EVT → DVT → PVT → MP

### 📊 项目部智能体（核心）
- **项目部文档入口**：8 份项目文件
- **输入输出文档**：明确显示项目部的输入文档（PRD、产品提案、MRD、竞品分析报告）和输出文档（8 份项目文档）
- **文档模板**：
  - 立项书（18 个字段）
  - 可行性报告（15 个字段）
  - 正式立项说明（8 个字段）
  - 项目章程（16 个字段）
  - 项目状态表（16 个字段）
  - 结项报告（13 个字段）
  - 成果报告（11 个字段）
  - 评审流程（12 个字段）
- **文档自动生成**：填写字段后自动保存
- **项目阶段追踪**：实时显示当前阶段

### 🔧 管理后台（完整权限）
管理员拥有所有增删改权限：
- **数据概览**：用户数、项目数、文档数统计
- **用户管理**：添加、编辑、删除用户
- **文档模板管理**：管理文档模板
- **字段管理**：增删改字段定义
- **流转规则管理**：配置项目阶段流转规则
- **同步规则管理**：配置文档间字段同步规则
- **部门管理**：管理部门架构

## 🚀 使用指南

### 1. 登录系统
- 打开上面的链接
- 使用管理员账号登录：`admin` / `Admin@2026`
- 或注册新用户

### 2. 创建项目（项目触发）
- 在首页填写项目名称、类型
- 输入项目描述/PRD 内容
- 上传文档（PRD、项目提案等）
- 选择负责部门
- 点击"创建项目"

### 3. 使用项目部智能体
- 点击"项目部智能体"入口
- 查看项目当前阶段（流程可视化）
- 查看输入输出文档（了解部门间文档流转）
- 选择文档类型（8 份项目文件）
- 填写表单字段
- 点击"生成文档"保存

### 4. 管理后台
- 点击"管理后台"
- 管理用户、文档模板、字段、流转规则、同步规则等

## 📊 项目流转流程

```
立项 → 规划 → 设计 → EVT → DVT → PVT → MP
 ↓      ↓      ↓     ↓     ↓     ↓     ↓
立项书  项目章程 设计文档 EVT 报告 DVT 报告 PVT 报告 结项报告
可行性报告        状态表
正式立项说明
```

### 流转规则

| 转换 | 条件 |
|------|------|
| 立项 → 规划 | 立项书已审批 |
| 规划 → 设计 | 项目章程已确认 |
| 设计 → EVT | 设计评审通过 |
| EVT → DVT | EVT 评审通过，无 P0/P1 问题 |
| DVT → PVT | DVT 评审通过，模具验收完成 |
| PVT → MP | PVT 评审通过，直通率达标 |

## 📁 文件管理系统（v7.0 新增）

### 核心功能

**智能体自动创建项目文件夹结构**：
- 项目创建时自动生成完整文件夹结构
- 文件夹以项目名称命名
- 智能体根据工作流程顺序自动编号（无需人工预设）

**文件夹结构示例**：
```
{项目名称}/
├── 00-项目公共/          ← 跨部门共享文件
├── 01-产品部/            ← 智能体根据流程自动编号
├── 02-项目部/
├── 03-软件部/
├── 04-硬件部/
└── 05-结构部/
```

**文件命名规范**：
```
{文档类型}_{版本号}_{创建者}_{日期}.{扩展名}

示例：PRD_v1.0_张三_20260317.docx
```
- 智能体自动补全文件名（用户只需选择文档类型）
- 版本号自动递增
- 日期格式：YYYYMMDD

**权限设计**：
- 智能体：可创建文件，**不可删除文件**
- 人工：可删除文件（移动到回收站）
- 回收站保留 30 天，支持找回，无需审批

**版本管理**：
- 文件夹内只显示最新版本文件
- 历史版本通过 Git history 查询
- 支持历史版本找回

**技术实现**：
- 当前：GitHub 文件系统
- 预留：飞书云空间接口（未来可扩展）

📖 **详细配置文档**: [docs/FILE_MANAGEMENT_CONFIG.md](docs/FILE_MANAGEMENT_CONFIG.md)
